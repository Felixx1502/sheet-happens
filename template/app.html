__TITLE__ — how this map was built

Three zoom levels

① Sheets — tabs and aggregate flows, laid out in calculation order. ② Blocks — auto-detected functional sections of each sheet. ③ Items — double-click any block: every line item (row), its formulas, and item-to-item links, including external feeders (left) and consumers (right).

Known limitations — read before relying on this map

  • Ranges: a reference like SUM(A5:A200) creates block-level links to every block the range crosses, but the item-level view links only the range's first row (marked as a dashed "range" link).
  • Dynamic references (INDIRECT, OFFSET targets, CHOOSE) cannot be traced statically — OFFSET maps to its anchor cell only.
  • Defined names are resolved from the workbook's name table at build time; names pointing to other workbooks or broken (#REF!) names are skipped (see Audit).
  • Structured table references (Table1[Col]) are counted in the Audit but not mapped to rows.
  • External workbook links are flagged in the Audit, not drawn.
  • Block boundaries are heuristic (section headers / blank gaps). Verify before quoting; you can pin them with a blocks_override.json.
  • Edge weight counts distinct formula-cells referencing the target block — thickness means "many live cells depend on this", not financial materiality.
